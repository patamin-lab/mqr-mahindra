# .claude/playbooks/

Step-by-step operational procedures for recurring multi-step work that
isn't a single coding task — deploys, onboarding a module, handling an
incident.

## Index

- `deploy-without-git-cli.md` — how changes actually get shipped when no
  local git CLI / clean clone is available
- `onboard-new-module.md` — the sequence for standing up one of the
  planned modules from `docs/MODULE_GUIDE.md`
- `incident-response.md` — what to check first when production breaks
